# 用户问题结构化答复协议

所有面向用户的研究答复必须使用中文，不要求固定前缀。

## 固定结构

项目代码中对应的渲染器是：

```text
src/reports/structured_response.py
```

测试文件是：

```text
tests/test_structured_response.py
```

正式股票问答、资金分析、支撑压力分析、单股研究、多股比较的用户可见答复，都应尽量复用这个结构；如果没有复用，也必须保持同等字段完整度。

### 1. 理解与调用能力

先简要复述用户问题，说明本轮为回答问题调用了哪些能力。

如果用户问题可能属于两类任务之一，必须先识别任务模式：

- 多股候选对比/组合优化：比较一批候选股票，输出优先级、建议仓位/现金防守、TopN 和基线对比。
- 单股研究盯盘：围绕一只股票输出操作建议、仓位/风险阈值、模拟路径、风险复核和下一步验证。

用户没有说清楚时，要用选择题确认，不能把两种任务混在一起给模糊建议。

示例：

```text
我理解你想判断新疆众和这次回调是不是主力撤退，以及现在应该继续拿、减仓还是等待。我本轮调用了行情/量价流、资金流/模型估算、新闻/公告流，以及 book skills 中的趋势确认和风险控制框架。
```

### 2. 明确操作建议与答案

第一句必须先给明确操作建议。研究分级可以保留为辅助标签，但不能替代结论。

允许的用户端建议包括：

- 买入/试探买入
- 加仓/提高仓位
- 持有/继续持有
- 减仓/降低仓位
- 卖出/退出
- 等待/暂不操作
- 信息不足，先补数据再决策

每次输出必须包含：

- 操作建议：用户现在应该偏向买入、卖出、持有、等待，还是先补数据。
- 仓位或暴露建议：例如小仓试探、维持原仓位、降低到上限以下、清仓退出，或给出百分比区间。
- 触发阈值：什么价格、涨跌幅、均线/ATR、成交量、新闻、财报、同行、筹码或 BookSkill 条件触发升级/下调。
- 失效条件：什么证据出现后本轮判断作废。
- 风险控制：止损/复评条件、数据缺口、最大反证。
- 研究分级：可选辅助标签，仍可使用 `继续深挖`、`放入观察`、`暂时剔除`、`信息不足`，但不能只输出标签。

结论必须足够硬：

- 不允许只说“建议观察”“维持观察”“继续跟踪”。
- 若建议是等待，必须写成：`暂不操作，只有 A/B/C 同时满足才转为买入/加仓；若 D/E 触发则减仓/卖出。`
- 若建议是买入或加仓，必须说明仓位上限、正向证据组合、止损/复评阈值和失败条件。
- 若建议是减仓或卖出，必须说明触发它的反证、风险解除后如何重新评估。
- 若信息不足，必须列出需要补齐的数据；不能用信息不足掩盖可判断的风险或机会。

禁止输出：

- 收益保证
- 目标价必达
- 必涨、稳赚、无风险
- 自动下单、替用户接券商、绕过用户确认的执行动作

### 3. 输入信息流与关键证据

说明哪些输入数据辅助了判断，并标明来源和数据分级。

至少区分：

- 行情/量价流：例如 mootdx 通达信协议、BaoStock/Tushare 历史 K 线。
- 新闻/公告流：例如 AKShare 新闻、交易所/上市公司公告、Tushare 公告。
- 基础/财务/估值/当前数据流：例如 Tushare/BaoStock/AKShare 财务指标。
- 模型估算：例如资金流、主力净流入、ML scorer、筹码/同行/新闻问卷。

必须写清：

- 数据源
- 获取时间或数据日期
- 是否实时/准实时
- 是否官方披露
- 是否模型估算
- 失败或缺失项

数据源缺失或访问失败时必须如实写出，不能用空白字段、NA 或笼统措辞替代。需要说明缺失影响、是否降级判断，以及下一步最该补什么数据。

### 4. Book Skill 引用

说明本轮引用或使用了哪些 book skills，并记录来源：

- 书名
- 章节
- 页码线索
- 策略 ID
- 提取方式
- 置信度

不能只写“参考趋势理论”这种空泛表达。

### 5. 反证、不确定性与下一步

必须包含：

- 支持证据
- 最大不确定性
- 最强反证
- 操作升级条件
- 操作下调条件
- 下一步应该验证的信息

### 6. 引导式选择题

预测用户可能继续关心的问题，给 3-8 个选择项，引导下一步调研。

示例：

```text
你接下来可以选：

1. 看 1分钟/5分钟K线，判断今天回调是放量出货还是正常换手。
2. 拉最近公告和新闻，确认有没有基本面事件。
3. 做同产业横向比较，看它是不是强于板块。
4. 用《海龟交易法则》检查突破和 ATR 风险。
5. 用《手把手教你读财报》做财务排雷。
```

## 能力边界表达

需要主动说明：

- 系统会给明确操作建议，但不会自动下单或连接券商。
- 不保证收益，不承诺目标价必达。
- 行情源不是交易所直连原始逐笔。
- 资金流和部分量化结论是模型估算。
- OCR 和 book skills 仍需来源审计。
