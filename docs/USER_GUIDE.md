本系统做 A 股研究辅助型决策建议：会给出清晰的买入、卖出、加仓、减仓、持有、等待或补数据建议，并配套操作阈值、反证、风险控制和复评条件。系统不会自动下单，不接券商接口，不保证收益。

## 建议到底怎么看

系统输出必须先给明确操作建议，再给依据和阈值，不允许只给一个模糊标签：

- `买入/试探买入`：必须给仓位上限、正向证据、止损/复评阈值和失败条件。
- `加仓/提高仓位`：必须说明为什么现有仓位可提高、提高到什么区间、什么条件下撤回。
- `持有/继续持有`：必须说明继续持有的条件，以及跌破或反证出现时如何处理。
- `减仓/卖出`：必须说明触发风险、退出阈值、风险解除后如何重新评估。
- `等待/暂不操作`：不能只写观察；必须写清什么条件转为买入/加仓，什么条件转为减仓/卖出。
- `信息不足`：必须列出最关键缺口，补齐前不硬做方向判断。

研究分级 `继续深挖`、`放入观察`、`暂时剔除`、`信息不足` 可以作为辅助标签保留，但不能代替操作建议。每条建议都必须配套：为什么、买入/加仓触发、减仓/卖出触发、下次复查条件。回测里会把这些建议转成内部模拟动作和权重，用来验证它是否真的提高 20 日表现或降低大亏。

## 启动向导

```bash
python -m src.user_wizard
```

如果用户的问题比较模糊，先让用户选择任务：

1. 单支股票盯盘/调研
2. 多股候选对比
3. 两者结合
4. 盘中/实时盯盘
5. 策略研究/组合回测

## 盘中盯盘

单次复核示例：

```bash
python scripts/run_live_watch_session.py --code 000001 --name 平安银行 --interval-seconds 1200 --max-iterations 1
```

连续盯盘时把 `--max-iterations` 改成需要的决策点数量。新闻公告默认按日缓存；如果实时行情、分钟线、新闻、公告或财报源失败，系统必须明确标注数据缺口，必要时输出 `信息不足`。

- P0 单支盯盘：强黄灯 MVP。当前默认交付“小仓试探/持有/减仓复核”，先由工具给小仓草案，再由 Agent 审计硬反证和软缺口。`PPS-Q-017 + softgap v2` 三面板 Flash 20 日正收益率 `0.8434±0.0501`、平均收益 `+1.9139±0.0990pp`；general-channel small-entry 三面板 Flash 正收益率 `0.8213`、平均收益 `+2.6015±0.5526pp`。Pro 可稳定调用但没有超过 Flash，因此只做高价值个案复核。

- 非价格信息：新闻、财报、同行/地域、BookSkill 目前主要用于解释、反证和防错杀。系统不会因为“同行弱”或“近期无财报事件”就机械剔除高质量回撤反转候选；但如果同类候选同时出现高风险新闻，会提示二次确认或降低研究置信度。
