# A股研究回测工作流用户说明

本说明面向使用者，而不是开发者。它解释这套研究系统能做什么、报告怎么看、哪些是基线、新闻 Agent 为什么重要，以及哪些结论不能越界。

## 一句话理解

这套系统是一个 A 股研究辅助型操作建议工具。它会把行情、新闻、公告、同行背景、Book Skill 和反证信息合在一起，帮助用户做两类研究：

- 多股票组合筛选：从一个候选池里分配研究注意力，比较 TopN、现金防守和多个基线。
- 单股票评估/盯盘：对某一只股票进行持续观察、排雷、买入/卖出/加减仓/持有/等待建议和模拟研究暴露调整。

系统会输出明确操作建议和阈值；`继续深挖`、`放入观察`、`暂时剔除`、`信息不足` 只作为辅助研究分级。

系统不会自动交易，不接券商接口，不下单，不承诺收益或目标价必达。

## 用户会看到什么

一次完整报告通常包括：

| 模块 | 用户看到的内容 | 用途 |
|---|---|---|
| 数据工作流 | 股票池、数据来源、决策频率、验证周期 | 确认测试是否可信 |
| 组合优化结果 | 优化后策略和多个基线的对比 | 判断多股票筛选是否有增量 |
| 单股评估结果 | 每股独立观察时的动作率、20日表现、现金对照 | 判断单股链路是否能过滤无效样本 |
| 新闻预警/机会提醒 | 风险预警数、机会提醒数、提醒后的20日表现 | 看新闻通道是否真的参与决策 |
| Ablation Test | 屏蔽某个组件后的表现 | 判断每个组件是否有贡献 |
| 系统性结论 | 用普通语言总结结果和限制 | 帮用户形成下一步研究动作 |

每次正式交互前，系统应先确认用户要做哪类任务：

1. 多股组合优化：用户给一批候选股票或一个行业/主题池，系统输出排序、分散度、现金防守比例和基线对比。
2. 单股研究盯盘：用户指定一只股票，系统输出操作建议、模拟路径、关键风险、新闻/公告变化和下一步复核项。

如果用户没有说清楚任务类型，KimiWork/Codex 端应先用选择题确认，而不是混合输出。

## 什么是基线

报告里的浅灰色行是基线或参考策略。它们不是我们的新模型，而是用来比较“新模型是否真的有用”。

常见基线包括：

| 基线 | 含义 | 为什么需要 |
|---|---|---|
| 原始系统Top3 | 没有新新闻 Agent 优化的上一版策略 | 判断优化是否真的提升 |
| 随机Top3基线 | 同一时间随机选3只股票 | 判断是不是随便选也能涨 |
| 全50等权基线 | 候选池所有股票等权持有 | 判断筛选是否优于不筛选 |
| 银行3%年化基线 | 不参与股票观察，按现金收益折算 | 判断是否连低风险现金都跑不过 |
| 原始辅助分级 | 只用继续深挖/放入观察等原始分级 | 判断操作建议流程是否比简单分级更有效 |

如果优化后策略没有持续跑赢这些灰色基线，就不能说系统有稳定增量价值。

## Top3 / Top10 是什么

Top3 和 Top10 都不是自动下单清单，也不是收益保证。

它们的含义是：在每个回测决策日，系统先对候选池里所有股票打分排序，然后取排名靠前的 N 只股票，做等权的研究模拟。例如候选池有50只股票，Top3就是当天分数最高的3只；Top10就是当天分数最高的10只。系统观察这些股票未来20个交易日表现，用来验证研究排序是否有效。

原始系统Top3是上一版对照策略。V2泛化Top10是在同一套研究边界下，把每个决策日入选数量从3只扩大到10只。它不是为了追求最近收益最大，而是为了降低日期过拟合：如果Top3只在近期好、2023不好，说明排序太尖锐；Top10更分散，可能牺牲部分收益均值，但更容易跨年份保持正收益率。

## NA 或“未到期”是什么意思

报告用未来 20 个交易日表现来验证每次研究决策。

因此，最近 20 个交易日内发生的决策还没有完整结果，不能提前算收益。这时表格会显示“未到期”。

还有一种情况是某个策略在某个周期没有触发，例如随机基线或系统策略在最近窗口没有可验证样本，也会显示未到期或无样本。

这不是程序错误，而是为了避免偷看未来。

## 新闻 Agent 怎么工作

新闻层不是简单判断“利好/利空”。它会把信息拆成四类背景和一组相对值：

| 新闻层级 | 用户理解 | 当前状态 |
|---|---|---|
| 股票自身新闻 | 这家公司自己有没有公告、订单、业绩、处罚、减持、诉讼等事件 | 已接入 |
| 领域/同行背景 | 同行业其他股票是否集中被提及，是否出现共性风险或机会 | 已接入基础版 |
| 政策/宏观背景 | 政策、补贴、价格、关税、利率、汇率等背景是否变化 | 已接入标题关键词版 |
| 地域背景 | 地方政策、区域产业链、区域监管、区域灾害等 | 预留，不编造 |

相对值很重要。比如：

- 自身新闻曝光高于同行：说明关注度升温，但不一定是好事。
- 同行频繁被提及但自身沉默：可能是信息滞后，也可能是公司没有跟上行业主线。
- 同行风险升高：即使个股自身没有新闻，也要小心行业风险溢出。
- 政策背景被提及：要结合公司在产业链中的位置，不能直接判定利好利空。

## 新闻提醒怎么读

| 提醒 | 含义 | 用户动作 |
|---|---|---|
| 风险预警 | 出现减持、处罚、诉讼、停产、冲突信息、同行风险等 | 先复核来源，再决定是否暂时剔除或信息不足 |
| 机会提醒 | 出现订单、中标、业绩、技术、关注度升温等 | 继续深挖，但必须看趋势、估值、Book Skill 和反证 |
| 无明显新闻提醒 | 当前新闻层没有提供增量信息 | 不能据此认为没有风险，只能说新闻证据不足 |

新闻提醒不是最终结论。它只是研究证据的一层。

## 模型工作流

1. 选择股票池：例如 50 股、200 股或更大的候选池。
2. 按决策日生成样本：每次只使用当日及以前可获得的数据。
3. 计算基础特征：趋势、回撤、均线、波动、相对强弱、财务可用性。
4. 计算新闻 Agent 表：自身新闻、同行背景、政策背景、相对曝光、风险预警、机会提醒。
5. 触发 Book Skill：只引用已有真实策略 ID，不修改原始策略卡。
6. 组合筛选：在候选池环境合适时，从候选池里选 TopN。
7. 单股评估：对单只股票做研究分级和盯盘优先级判断。
8. 未来验证：用未来 5/10/20 个交易日表现检查研究决策质量。
9. 做 ablation：屏蔽新闻、趋势、环境 Gate 等组件，验证每个组件是否有用。
10. 输出用户报告：用基线、指标、路径和限制说明结果。

## 能力边界

这套系统擅长：

- 发现候选池里的相对强弱和阶段性机会。
- 把新闻、同行、政策背景转成可比较的研究信号。
- 用 ablation 判断某个组件是否真的有贡献。
- 帮用户减少无效观察，把研究精力集中到更值得看的股票上。

这套系统不擅长或当前还没有完成：

- 不预测确定收益。
- 不做自动交易。
- 不处理真实下单、滑点、成交限制。
- 当前还没有完整逐日净值曲线、最大回撤和交易成本模块。
- 地域新闻背景目前只是预留字段，没有可信数据时不会强行生成。
- 免费新闻/公告聚合源不等于官方原文闭环，重大事件必须人工复核。

## 数据缺失和访问失败怎么处理

系统不会把缺失数据当作默认利好，也不会为了凑结论而编造来源。

用户报告必须直接说明：

- 哪个数据源失败，例如新闻、公告、财务披露日、Book Skill 来源或日线缓存。
- 失败影响的是组合模式、单支模式，还是两者都有影响。
- 当前结论是否因此降级为“信息不足”或“放入观察”。
- 后续最应该补哪类数据。

如果某个通道在回测中没有证明贡献，报告应写成“当前未证明有效”，不能写成系统优势。

## 如何判断一轮结果是否好

用户可以按这个顺序看：

1. 优化后策略是否跑赢原始系统Top3。
2. 是否跑赢随机Top3、全池等权和现金基线。
3. 20日均值是否提高，正收益率是否稳定，波动和大跌比例是否可接受。
4. 全样本是否仍然有效，而不是只在最近12个月有效。
5. Ablation 中去掉新闻、趋势、环境 Gate 后是否明显变差。
6. 报告是否清楚说明样本不足、未到期和数据缺口。

只有这些条件同时较好，才能说系统在研究层面有增量价值。

## 本轮判断工作流程

这轮不是直接追求最近窗口最高收益，而是优先解决日期过拟合。

判断顺序如下：

1. 先检查数据边界：当前500股缓存没有2022日线，所以2022不能验证，只能标为无样本。
2. 固定外部股票：剔除 `backtest_scale_200` 已参与训练/调权的股票，只用三组未见50股面板。
3. 只用2024-2026比较候选策略，不用2023及以前调参。
4. 发现原始系统Top3在2023未训练验证中正收益率不足0.6。
5. 比较少量预设TopN方案，避免复杂搜索导致过拟合。
6. 选择V2泛化Top10，因为它在2023未训练验证中正收益率超过0.6，且20日均值相对原始Top3提升超过0.2个百分点。
7. 同时承认代价：Top10更分散，训练期部分年份的20日均值低于Top3，所以它是泛化优先方案，不是收益最大化方案。
8. 检查新闻层：当前未见面板新闻覆盖不足，不能把V2提升归因于新闻；新闻 Agent 的表结构和产品设计已完成，但需要补充可回测新闻源才能证明量化增量。

## 产品级外部评估怎么读

正式产品评估不只看一组50股。当前更严谨的评估方式是：

- 从500股缓存中剔除 `backtest_scale_200` 已参与训练/调权的全部股票。
- 从剩余未见股票中构造三组互不重叠的50股面板，共150支。
- 分别评估2022未训练验证、2023未训练验证、2024全年、2025全年、2026年初至今、近6个月、近12个月和全样本。
- 对三组面板计算均值和标准差，避免单一样本偶然性。
- 当前缓存最早从2023年开始，没有2022日线，因此2022窗口会显示无样本；这不是策略通过或失败，而是数据不足。

产品评估数据位置（PDF 不再作为交付入口）：

- `reports/product_multi_panel_eval/aggregate_metrics.csv`
- `reports/product_multi_panel_eval/panel_metrics.csv`
- `reports/product_multi_panel_eval/news_panel_metrics.csv`
- `reports/product_multi_panel_eval/product_eval_panels.yaml`

单组50股案例数据位置：

- `reports/external_50_eval/portfolio_metrics.csv`
- `reports/external_50_eval/single_stock_metrics.csv`
- `reports/external_50_eval/news_alert_summary.csv`
- `reports/external_50_eval/ablation_metrics.csv`

报告中的灰色行是基线。用户应优先比较“优化后新闻感知Top3”和这些灰色基线。

当前更严谨的产品结论应这样表述：

- V2泛化Top10在未见股票三面板上用于检验日期泛化：不使用2023及以前调参，只用2024-2026选择方案，再看2023未训练验证。
- 2023未训练验证里，V2需要20日正收益率超过0.6，并且相对原始系统Top3的20日均值提升至少0.2个百分点，才算达成本轮泛化目标。
- 优化后策略还需要同时参考随机Top3、全50等权和现金基线，确认不是随机选择或全池上涨带来的假象。
- 相比原始系统Top3，优化后策略不是每个窗口都提高20日均值；必须看均值、标准差和稳定性，而不是只看单一窗口。
- 未见三面板当前新闻提醒覆盖接近0，说明免费缓存新闻不足以证明新闻层的跨年量化增量。
- 新闻 Agent 的产品设计已经明确：政策背景、领域/同行背景、地域背景、股票自身新闻和相对曝光差都应进入量化表；但要证明它有效，后续必须补充可回测、时间安全的新闻源。
- 新闻机会提醒必须和环境 Gate、长期趋势保护、Book Skill、反证一起使用，不能独立形成研究结论。
