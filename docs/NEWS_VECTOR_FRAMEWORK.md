# 新闻向量化框架

## 目标

把新闻、公告、行业事件转成可回测、可审计、可替换的结构化特征。该模块本身只输出事实和结构化特征，不直接给买入/卖出/加减仓结论；最终操作建议由 Agent 综合多通道证据生成，且不得承诺收益或目标价必达。

## 时间窗约束

- 决策日只使用当日 15:00 及以前已可得的信息。
- 有精确 `发布时间` 的新闻，按分钟级时间戳过滤；例如 `2026-06-23 14:30:00` 可用于 2026-06-23 收盘后决策，`2026-06-23 16:30:00` 不可用于当天决策。
- 只有日期、没有具体发布时间的公告，默认从下一自然日开始可用，避免同日偷看。
- 新闻缓存缺失时输出全 0 向量，不阻塞行情和 Book Skill 回测。

## Kimi/Agent 分类 Schema

Kimi 或其他 Agent 处理新闻正文时，必须输出下面这些清晰字段；字段缺失时用 `不确定` 或 0，不允许脑补来源。

| 字段 | 取值/范围 | 说明 |
|---|---|---|
| `event_title` | 文本 | 原始标题，不改写事实 |
| `available_at` | `YYYY-MM-DD HH:MM:SS` 或日期 | 信息可用时间；只有日期的公告按下一日可用 |
| `source_type` | `官方公告` / `公开新闻` / `行业新闻` / `传闻/未证实` | 官方公告优先级最高 |
| `entity_scope` | `公司` / `行业/政策` / `上游` / `下游` / `市场` / `不确定` | 事件作用范围 |
| `event_type` | `业绩` / `订单` / `产能` / `增减持` / `法律监管` / `融资` / `价格政策` / `技术产品` / `宏观市场` / `供应链` / `一般新闻` | 后续映射到 32 维向量 |
| `direction_hint` | `正面` / `负面` / `中性` / `不确定` | 只表达研究影响方向，不表达买卖建议 |
| `materiality_score` | 0-10 | 是否足以改变研究结论；7 以上视为重大 |
| `evidence_level` | `official_disclosure` / `public_aggregator` / `model_inferred` / `manual_review_required` | 证据层级 |
| `conflict_flag` | `无冲突` / `正负冲突` / `来源冲突` / `时间冲突` | 用于反证 |
| `summary_for_vector` | 1-2 句 | 只摘要事实，不直接给操作建议 |

分级建议：

- `materiality_score >= 8`：监管处罚、重大诉讼、重大减持、重大停产、业绩大幅预减、重大订单/中标、重大资产重组。
- `materiality_score 5-7`：一般业绩改善、产能投放、融资、价格政策变化、重要客户/产品进展。
- `materiality_score 1-4`：普通报道、资金流、人物任命、重复转载、低相关行业新闻。
- `direction_hint=正面` 不等于提高分级；必须结合价格位置、同组状态、Book Skill 和反证。
- `direction_hint=负面` 默认进入风险/反证通道，除非后续公告澄清。

## 当前 32 维向量

当前实现位于 `src/backtest/news_vector.py`，维度固定为：

- 数量与来源：新闻数、官方公告数、公开新闻数、公司事件数、行业/政策事件数。
- 方向：正面、负面、不确定数量。
- 重大性：重大性总和、最大值、正面重大性、负面重大性、官方重大性、公开新闻重大性、净重大性、时间衰减重大性。
- 事件桶：业绩、订单、产能、增减持、法律监管、融资、价格政策、技术产品、宏观市场、供应链。
- 组合信号：风险事件分、机会事件分、证据质量分、冲突强度、事件类型多样性、最高事件重大性。

## 后续可升级方向

- Agent/LLM 分类：把标题和正文分类到相同 32 维 schema，保留人工复核字段。
- 语义嵌入：在 32 维结构化特征外增加 embedding，用于相似案例检索。
- 关键词频：保留事件桶关键词频率，辅助解释为什么某条新闻影响分级。
- 相关股票传播：把同产业链或同板块公司的高重大性新闻聚合成 peer/news spillover 向量。

## 自动化来源优先级

| 优先级 | 来源 | 当前接入状态 | 用途 | 风险 |
|---|---|---|---|---|
| P0 | 东方财富个股新闻 / AkShare `stock_news_em` | 已接入 `scripts/prepare_backtest_news.py` | 最近新闻、标题、正文片段、发布时间、来源、链接 | 历史覆盖偏近期，需防止把近期市场环境误当新闻因果 |
| P0 | 东方财富公告大全 / AkShare `stock_notice_report` 底层接口 | 已接入为可选源 `eastmoney_notice` | 个股公告、公告类型、公告日期、链接 | 半年全量 200 股较慢，建议分批增量 |
| P1 | 上交所/深交所/巨潮公告 | 待接入 | 官方公告校验和补漏 | 页面/API 结构可能变化，需要缓存和失败跳过 |
| P1 | 财联社/证券时报/e公司等快讯 | 待接入 | 市场/行业事件、政策、主题热度 | 来源许可和接口稳定性需单独验证 |
| P2 | 研报/券商评级/互动易问答 | 暂不接入 | 辅助研究背景 | 噪声与滞后较高，且容易引入主观观点 |

采集原则：

- 公告优先于新闻，官方披露优先于公开聚合。
- 新闻采集脚本必须记录 provider、source、url、datetime/date。
- 单个源失败只记录错误，不中断回测。
- 历史回测使用新闻时，必须先确认该事件在决策日之前可得。

## 使用边界

新闻向量只能作为操作建议、辅助研究分级和反证风险的输入。任何由新闻向量挖掘出的阈值，都必须同时记录样本数、验证窗口、test 表现和失败样本；不得由新闻单通道直接触发买入/加仓。
