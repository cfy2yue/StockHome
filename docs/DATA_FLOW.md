# 输入数据流设计

本项目的数据输入拆为四路：行情/量价流、新闻/公告流、财报/业绩公告事件流、定量/当前数据流。每一路都要标明来源分级、是否实时、是否官方、是否模型估算。

## 1. 行情/量价流

目标：尽量接近行情软件里的量价体验，支持准实时行情、当日 K 线、分钟线、历史日线。

优先级：

1. `mootdx`：通达信行情服务器，优先用于准实时行情、盘口、当日日线、1 分钟/5 分钟 K 线。
2. `BaoStock`：免费结构化历史数据，优先用于历史日线、指数和回测。
3. `AKShare/efinance`：公开聚合接口，作为补充和备用。
4. 本地缓存：接口失败时降级。

可用字段：

- 准实时行情：现价、昨收、开盘、最高、最低、成交量、成交额、买一卖一至五档等。
- 当日/历史 K 线：开、高、低、收、成交量、成交额。
- 分钟线：1 分钟、5 分钟等周期 K 线。

边界：

- mootdx/pytdx 是通达信协议数据，不是交易所直连原始逐笔数据。
- 免费源可能延迟、断连或缺字段。

## 2. 新闻/公告流

目标：覆盖个股新闻、公告、行业事件和市场事件。

优先级：

1. 官方披露：交易所、巨潮、上市公司公告源。
2. AKShare/东方财富新闻：个股新闻、市场新闻。
3. 本地缓存：接口失败时保留最近可用结果。
4. 用户合法授权的 `paid_standardized` 离线缓存：Tushare Pro、Wind、Choice、iFinD、同花顺会员等；只允许代码读取本地 token/key，不得输出明文。

输出必须区分：

- 事实
- 解释
- 猜测
- 待验证事项

## 3. 财报/业绩公告事件流

目标：把定期报告、业绩预告、业绩快报、修正公告、审计意见、问询回复等作为高可信新闻类事件，同时保留报告期和披露日，避免把未来财报提前用于回测。

优先级：

1. 官方披露：巨潮资讯、上交所/深交所公告、上市公司 PDF 原文。
2. `paid_standardized` 离线缓存：Tushare Pro/Wind/Choice/iFinD/同花顺会员等，必须保留 `ann_date/f_ann_date`、报告期、接口名和来源类型。
3. 公开聚合源：AKShare/东方财富/efinance 只能补漏和交叉检查；缺披露日时不得进入 walk-forward 判断。

当前工程入口：

- `src/world_model/financial_report_channel.py`
- `scripts/build_financial_report_channel.py`
- `data/date_generalization_cache/market_5000/financial_report_events.csv`
- `data/date_generalization_cache/market_5000/financial_report_features.csv`
- `reports/date_generalization/financial_report_channel_coverage.md`

当前实测覆盖：

- 4423 条财报/业绩公告事件。
- 1470 条股票-日期特征。
- 覆盖 1337 只股票。
- 90 天 as-of GT 匹配率 3594 / 155690 = 0.0231。
- 本地分层回测 `financial_report_local_stratified_analysis`：时间安全匹配样本 3558 条、110 只股票。高财务质量风险组正向率低于基线 0.2400、亏损率高于基线 0.2385，但只覆盖 3 只股票；多报告复核触发器覆盖 1020 条、40 只股票，正向率小幅高于基线 0.0394、亏损率低于基线 0.0356。

边界：

- 只有日期、没有分钟级发布时间的披露，按下一自然日 00:00:00 可用，避免同日收盘前偷看。
- 缺少披露日、报告期或关键字段时，只能进入缺失/不确定性，不能作为正向或负向判断。
- 当前只证明通道可用、时间安全，以及更适合复核/风险/不确定性；尚未证明可作为独立正向收益来源。后续必须做财报通道消融，并控制股票集中度。

## 4. 定量/当前数据流

目标：支持财务排雷、估值、安全边际、横向比较，以及正式研究时的当前定量状态描述。

优先级：

1. BaoStock：历史结构化行情、财务指标、估值辅助指标。
2. AKShare：财务摘要、资产负债表、利润表、现金流、行业板块。
3. 官方财报/公告：用于最终复核。

边界：

- 银行、券商、保险等金融股不应套用普通制造业财务排雷规则。
- 估值和财务指标需要标注报告期。
- 当前数据若来自公开聚合源，必须标注更新时间和接口口径。

## 5. 资金流数据

资金流单独标为 `model_estimate`。它可以用于观察情绪和资金偏好，但不能当作交易所事实。

## 6. 报告要求

报告中每个模块必须展示：

- 数据源名称
- 数据源分级
- 获取时间
- 成功/失败状态
- 是否影响结论
- 是否为模型估算

## 7. Skill Bridge 增强层（补充数据流）

系统在后台自动尝试调用 Kimi Work 内置 `stock-assistant` 系列 skill 作为补充数据源，用户无需感知，失败时静默回退到现有主数据流。

当前增强项：

| 增强项 | 调用方式 | 补充内容 | 触发条件 |
|---|---|---|---|
| 技术指标补充 | `stock_atom bundle technical` | MA/RSI/MACD 等完整技术指标 | 单股分析，非 dry-run 模式 |
| 舆情搜索 | `stock_atom research search` | 最新新闻、公告、研报摘要 | 单股分析，非 dry-run 模式 |
| 多股快照 | `stock_atom quote snapshot` | 批量报价快照，用于横向对比 | 多股比较，非 dry-run 模式 |

配置位置：`config/skill_bridge.yaml`

行为：
- 默认开启，可通过 `enabled: false` 全局关闭。
- 单个增强项可独立关闭。
- 失败时自动降级，不阻塞现有流程，不弹窗提示用户。
- 结果以 "补充数据流" 形式写入 `stock_report.md`，以独立行写入 `data_status.md`。
- 来源分级为 `skill_bridge`，与现有 `quote_protocol`、`public_aggregator` 等并列。

边界：
- Skill 增强层依赖 Kimi Work 环境；在纯 conda/codex 独立运行时自动标记为 "未激活"。
- 所有 Skill 数据均作为补充参考，不改变现有评分权重和 book skills 判断逻辑。

## 8. 当前实测结论

- BaoStock 历史日线：可用，`history_daily` 字段完整。
- BaoStock 财务指标：可用，适合报告期财务指标，不是实时估值面板。
- mootdx 准实时行情、日线、1分钟/5分钟K线：可用，字段完整；属于通达信行情协议，不是交易所直连逐笔。
- AKShare 个股新闻：可用，但本轮字段缺少明确来源字段，报告中需标“公开聚合源，需交叉验证”。
- AKShare 个股公告：可用，但来源是东方财富公开聚合页，不等于交易所/巨潮官方原文闭环。
- AKShare 当前个股信息：本轮超时，已降级到 mootdx 当前报价；因此当前市值、PE/PB、行业字段本轮信息不足。
- efinance 东方财富接口：当前不稳定，暂不作为主流程依赖。
- 资金流：尚未纳入正式 smoke；未来接入时必须标为 `model_estimate`，不能当交易所事实。
- 财报/业绩公告事件流：Tushare Pro 本地离线缓存已可派生 4423 条时间安全事件并进入 evidence pack，但 90 天 as-of 匹配率只有 0.0231，暂作高可信辅助/不确定性通道，不宣称 alpha。

最新实测报告见：`reports/latest/multisource_data_smoke.md`。

## 9. 外部审查后的优化清单

P1：

- 公告流要继续补交易所/巨潮/上市公司官方原文链接，至少输出标题、公告日、来源 URL、是否官方披露。
- 行情 smoke 不只看记录数，已经增加字段完整性；后续还要校验是否包含最近交易日或盘中更新。
- 定量当前数据要补当前报价、市值、PE/PB、行业、报告期财务摘要、估值口径。

P2：

- `MultiSourceDataAdapter` 需要继续加入统一缓存、重试、超时和降级记录。
- 单元测试从“文档含 tier 名称”升级为 monkeypatch 离线测试，覆盖失败跳过和字段完整性。
- 报告中始终区分：官方披露、公开聚合、协议行情、模型估算、本地缓存。
